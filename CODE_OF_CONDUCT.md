# Code of Conduct

## Expected Behavior

Participants are expected to be respectful, constructive, and focused on improving the project.

Examples of expected behavior:

- use welcoming and professional language
- respect different experience levels and viewpoints
- keep feedback specific and actionable
- accept maintainers' scope decisions

## Unacceptable Behavior

Unacceptable behavior includes harassment, personal attacks, discriminatory language, sustained disruption, or publishing private information without consent.

## Enforcement

Maintainers may remove comments, close issues, reject contributions, or block participants whose behavior is harmful to the project or community.

## Scope

This code of conduct applies to project spaces including issues, pull requests, discussions, and related community channels.
